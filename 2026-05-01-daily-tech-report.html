WordPress MCP Adapter v0.5.0 — Typed Protocol DTOs

Tool

The official WordPress MCP Adapter received a significant internal upgrade in v0.5.0, adopting wordpress/php-mcp-schema across the plugin. MCP responses now use typed protocol DTOs instead of hand-built arrays, improving reliability and consistency. The original Automattic/wordpress-mcp repo is officially deprecated in favor of WordPress/mcp-adapter. The Abilities API is also moving into WordPress core as of version 6.9.

Why it matters: Typed DTOs mean more predictable responses when Claude Code talks to your WordPress sites via MCP. If you're using the older Automattic package, migrate now. With the Abilities API heading into core, the MCP bridge will become a first-class WordPress citizen — this is the infrastructure that makes Claude Code + WordPress a production-ready combo.
via GitHub

WordPress 7.0 Revised Schedule — Release Now May 20

Important

The WordPress core team pushed the 7.0 release from April 9 to May 20 due to architectural stability concerns with the Real-Time Collaboration (RTC) database layer. The revised timeline: RC3 (treated as beta) on May 8, RC4 on May 14, code freeze May 19, general release May 20. This is an unprecedented return from RC to beta status. WP 7.0 brings real-time co-editing, Breadcrumbs and Icons blocks, responsive Grid block, Gallery lightbox, a Web Client AI API, and refreshed admin design.

Why it matters: You have 19 days to test your client sites, Bricks Builder configurations, and custom plugins against WP 7.0. The RTC feature changes the database architecture — start testing on staging environments now, especially if you use custom editor extensions or collaborative workflows. The new Web Client AI API could open up native AI integration possibilities.
via Gutenberg Times

Gutenberg 23.0 Released — Template & Pattern Revisions

Update

Gutenberg 23.0 is available for download. The headline feature: editing a template, template part, or pattern now surfaces the same Revisions panel previously available only for posts and pages. Why it matters: Template revisions are a game-changer for agency work — you can now track and roll back changes to templates and patterns, not just posts. This reduces the risk of breaking a client's site layout during updates and adds an audit trail for template modifications.
